# TanukiParser 機能仕様書 (Functional Specification)

## 🐾 はじめに
本ドキュメントは、EBNF（拡張バッカス・ナウア表記）からパーサーコードを自動生成するシステム「TanukiParser」の機能仕様および設計背景を記述したものです。

## 🐾 プロジェクトの経緯 (History)
TanukiParserは、ご主人様がかつてC#の学習を兼ねて開発されたレガシープロジェクト**「OEBNF」**を、現代的なプログラミング言語である **Rust** へと完全移植・近代化したものです。

### 継承された資産
*   **文法仕様**: NTT独自仕様を含む拡張EBNFのパースロジック。
*   **構造**: 字句解析 -> 構文解析 -> 参照解決 -> コード変換 -> コード生成 という堅実なパイプライン。
*   **埋め込みコード**: `@{ ... @}` 形式によるセマンティックアクションの注入。

### 進化したポイント
*   **型安全性の向上**: C#のクラス継承ベースのASTから、Rustの代数的データ型（`enum`）による厳格なASTへと進化。
*   **マルチターゲット化**: 単一の文法ファイルから **C#, Rust, Python** のソースコードを生成可能。
*   **強力な診断エンジン**: 
    *   `Span` (行・列) に基づく正確なエラー報告。
    *   タイポや未定義ルールがあっても、生成コードのビルドを壊さない「耐故障性」。

## 🐾 主要機能 (Key Features)

### 1. EBNF構文解析
以下の要素を含むEBNF定義をパースします。
*   **定義**: `name = elements`
*   **連鎖・選択**: `|` (または空白区切り)
*   **反復・オプション**: `*`, `+`, `{ }`, `[ ]`, `( )`
*   **埋め込みコード**: `@{ ... @}` によるアクション。
*   **メタデータ**: `@namespace`, `@class`, `@decl`, `@token` 等の設定。

### 2. マルチターゲット・コード生成

#### Python ターゲット [NEW]
*   **モダンな実装**: Python 3.10+ の `match/case` 構文をフル活用。
*   **アクション連携**: `@{ ... @}` 内で `self.results` 等の動的なデータ操作が可能。

#### Rust ターゲット
*   **Rustらしい設計**: 戻り値に `Option<Node>` を採用。
*   **リソース効率**: `Tokenizer` を参照（`&mut`）で保持するライフタイム設計。

#### C# ターゲット
*   **互換性重視**: 従来のパーサーと同様の構造を出力。

### 3. 診断エンジン (Diagnostic Engine)
解析中に発生したエラーを収集し、一度に報告します。
*   **位置特定**: ソースファイル内の正確な位置を表示。
*   **コード保護**: 未定義参照箇所を自動でコメントアウトし、パーサーのビルドエラーを回避。

## 🐾 システム構成 (Architecture)

1.  **Tokenizer**: EBNFをスキャン。
2.  **EBNFParser**: 文法ASTを構築。
3.  **ReferenceResolver**: ルール名の依存関係を解決。
4.  **NodeParser**: 出力コードの論理構造（IR）へ変換。アクションの挿入タイミング（トークン消費直前）を制御。
5.  **CodeGenerator**: 各言語の具体構文を出力。

## 🐾 使用方法 (Usage)

```powershell
# Pythonパーサーを生成する場合
cargo run -- -i <input.ebnf> -o <output.py> --target python

# Rustパーサーを生成する場合
cargo run -- -i <input.ebnf> -o <output.rs> --target rust
```
