# TanukiParser 言語リファレンス (Language Reference)

TanukiParserが処理する独自EBNF（拡張バッカス・ナウア表記）の仕様を詳しく解説します。

## 🐾 文法の基本構造
文法ファイルは、大きく分けて「設定セクション（メタコマンド）」と「文法定義セクション（@@以下）」の2つで構成されます。

```ebnf
# 設定セクション
@namespace MyCompiler
@class MyParser

# 文法定義セクション
@@
root = expression
...
```

---

## 🐾 基本構文 (Core Syntax)

### 1. ルール定義 (`=`)
`ルール名 = 要素` という形式で定義します。
```ebnf
stmt = expression
```

### 2. 連結 (Concatenation)
要素を並べることで、順番に出現することを意味します（空白または改行で区切ります）。
```ebnf
pair = "(" ")"
```

### 3. 選択 (`|`)
いずれかの要素が出現することを意味します。
```ebnf
choice = "a" | "b" | "c"
```

### 4. オプション (`[ ]`)
要素が0回または1回出現することを意味します。
```ebnf
optional_sign = [ "-" ]
```

### 5. 反復 (`*`, `+`)
*   `*` (Repetition 0+): 0回以上の繰り返しを意味します。
*   `+` (Repetition 1+): 1回以上の繰り返しを意味します。
```ebnf
list = item { item }  # または item+ と記述可能
```
※TanukiParserでは `{ }` も0回以上の繰り返しとして扱います。

### 6. グループ (`( )`)
演算の優先順位を制御します。
```ebnf
grouped = ( "a" | "b" ) "c"
```

---

## 🐾 リテラルと特殊表記 (Literals)

TanukiParserは、文字列以外にも様々な数値リテラルをサポートしています（NTT拡張仕様）。

*   **文字列**: `"text"` または `'text'`
*   **2進数**: `%b0101`
*   **10進数**: `%d123`
*   **16進数**: `%x1F`
*   **散文（Prose）**: `< description >`（具体的な実装を持たない、人間向けの注釈。`<` と `>` で囲みます）

---

## 🐾 メタコマンド (Meta Commands)

生成されるコードの構成を指定します。

| コマンド | 引数 | 説明 |
| :--- | :--- | :--- |
| `@namespace` | 識別子 | 生成コードのネームスペース（C#）を指定します。 |
| `@class` | 識別子 | 生成されるパーサークラスの名前を指定します。 |
| `@decl` | 型 名 | パーサークラスのメンバ変数を宣言します。（例: `@decl list results`） |
| `@token` | 名 [リテラル] | 外部トークンを定義します。リテラルを指定すると、字句解析器が自動でその文字列をトークンとして認識します。 |

---

## 🐾 セマンティックアクション (Embedded Code)

`@{ ... @}` ブロックを使用することで、ルールがマッチした際に実行されるターゲット言語（Python / Rust / C#）のコードを直接記述できます。

### 特殊変数（ターゲット言語に依存）
*   **Python**: `self.token` (現在のトークン), `self.results` (蓄積用リスト)
*   **Rust**: 現在のノードツリー構築ロジックが優先されます。
*   **C#**: `$$`, `$` 等のエイリアス（将来的な完全互換性のため）。

```ebnf
# 例: パースした内容をリストに追加する（Pythonターゲット）
text_content = TEXT @{ self.results.append(self.token.content) @}
```

---

## 🐾 ベストプラクティス
1.  **ルールの命名**: ルール名は英数字で記述し、一目で役割がわかるようにしましょう。
2.  **トークンの分離**: 演算子やキーワードなどは `@token` で明示的に定義すると、パーサーの可読性が高まります。
3.  **アクションの活用**: `@{ ... @}` 内でデータを処理することで、単なる構文解析を超えたデータ抽出ツールとして活用できます。
